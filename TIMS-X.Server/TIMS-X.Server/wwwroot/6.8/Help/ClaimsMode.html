    Claims Mode

    In this mode, claims that have been generated, but not yet transmitted are displayed on the patient tree.

    Patient Tree

    Expanded:

    Collapsed:

    If the tree is collapsed, expand by clicking on the arrow symbol next to the patient name. 

    1. The First Level of the tree will display the Patient Name.  
    2. The Second Level displays the type of Claim, Claim Number, Total Charges, Appointment Site, and a "P" (professional claim) or "I" (institutional claim):
      1. New - claims consisting of newly added transactions that have not been submitted
      2. to any payer yet
      3. Pri - claims consisting of transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the primary payer.
      4. Sec - claims consisting of transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the secondary payer.
      5. Ter - transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the tertiary payer.
      6. Quar - transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the quaternary payer.
      7. Quin - transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the quinary payer.


    If the claim is determined to have errors that will prevent it from being accepted by the clearinghouse, it will be in red font. Hovering the mouse over the line will popup a tooltip displaying the errors.  Errors must be corrected before the claim can be transmitted.

    If the claim is in dark gold font, that means the total of all charges is zero.  It can still be transmitted with a zero balance, but may be rejected.

    Claim Information

    Click on a claim line to select it and see the details in the Claim Information section to the right.

    • Patient and Insurance Information section displays the patient name, appointment site, and date of service, along with the  insurance payers for the claim.  

      • Clicking the Patient Insurance button will open the Patient Insurance Editor screen, where you can edit the patient demographics and/or payer information.
      • The Current Claim Payers section will show which payers are currently assigned to the claim.  If the claim has been resubmitted, the payers may be different from the current Patient insurance if that has changed since the original transmission. At the time of re-submission, there is a choice to keep the current payer or switch to the new insurance.  The default is the current payer. 
        • If you wish to switch all the payers on the claim to the current payers as entered in patient insurance, click the down arrow button.  
        • A confirmation message will popup to verify that you wish to switch the claim payers.  This action is not reversible.  Click Yes to switch, No to cancel.
      • Clicking the Skip Payer button will bypass the primary payer and send the claim directly to the secondary.
    • The Diagnosis Codes section will display all diagnosis codes included with the claim.

    • The Patient Release/Assignment Signature information can be added/changed here or in the Patient details.

    • To indicate that additional documentation will be sent with the claim, enter information in the Claim Attachment Information section. Choose the type of document to be sent, and how it will be sent to the payer.  If the practice is setup to do so, the attachment can be sent through TIMS after the claim is transmitted.  (Contact Support at 800-763-0308 for details) See Sending a Claim Attachment in Transmitted Mode.

    • If additional information is needed for the claim, click the down arrow next to Expand Fields in the Additional Claim Information section. 

    • If a different Tax and/or NPI number than the default practice information is needed for the claim, click the down arrow next to Submitter Information.  Submitter choices need to have been previously entered in Setup.

    • If needed, the Service Location, Rendering Provider, Referring Provider, and or Supervising Provider can be changed here.

    • Double-click on a transaction line to open the Claim Transaction detail window.  Here you can add, edit or view transaction information. 

    Action Buttons

    The following buttons appear on top of the tree when in Claims mode:

    • Expand - expands all nodes on the tree, so that all levels are displayed for all patients.
    • Collapse - collapses the tree to a condensed state, so that only the top level (patient name) is displayed
    • Select All - places a check mark next to all transactions in the tree
    • UnSelect - removes the check mark from all transactions in the tree
    • Transmit - transmits all selected (checked) claims to the clearinghouse.  If no claims have been selected, no action will be performed.
    • Revert  - removes all the transactions from a claim in order to start over. This will make all the transactions available again and delete the current claim. You can then delete any or all of the transactions, edit them however you wish, or select one or more of them to rebuild a new claim. To revert claims with errors, right-click on the claim line and a popup message will ask if you wish to revert the claim.

    Processing Buttons


    • Prof/Inst Claim - changes a claim to be processed as Institutional type versus Professional (if practice is setup for Institutional claims).
    • Transfer Adjudication - moves the selected claim to the Adjudication link without transmitting to the clearinghouse first.  Claims will be marked as Manual Claims in Adjudication.  From Adjudication, a  claim form can be printed, if needed.  This function can be used to bypass the Primary payer and go to the Adjudication link for secondary processing. A popup message will confirm that you don't want to transmit the claim.

    • ClearingHouse - view previously received claim responses and download a response file without first transmitting a batch. See Reading Response Files and Download Response Files for more details.

    • View SuperBill Notes - if the transaction was initially created on an Electronic Superbill, any notes entered in that screen can be viewed here.
    • View Payer Notes - if the transaction has been previously submitted, any previous clearinghouse and payer claim status notes can be viewed here. 

      • Additional manual notes can also be added in this screen. See Payer and User Notes for more information.
