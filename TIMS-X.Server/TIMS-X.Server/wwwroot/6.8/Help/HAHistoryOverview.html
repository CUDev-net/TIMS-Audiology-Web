    HA History Overview

    The Hearing Aid History link contains current and past hearing device history information for the patient. This link also tracks a patient’s hearing aid orders, returns, repairs, loaners, implant components, and battery clubs.


    Hearing Aid History Summary Grid


    The hearing aid history grid displays summaries of hearing aid, earmold and accessory histories.

    • Patient name is displayed in the top left corner of the hearing aid history grid.
    • Use the check boxes next to Show Hearing Aids, Earmolds or Accessories to display only specific history types.
    • Click the Include Inactive checkbox to view inactive HA histories. (those with a Status that is not of type Active or Active w/ Threshold)
      • Active hearing aids have a status of type Active or Active w/ Threshold. By default, these are In Use, Purchased Elsewhere, Trial Period, On Order, Received and In Use-Repair Replacement, although custom "active" statuses may be used as well.
    • Click the Refresh button to update the grid to include any changes that have been made a hearing aid history.

    Hearing Aid History Header

    1. The Type of history - Hearing Aid, Earmold, or Accessory
    2. Purchase Date of the device.  If Purchase Date is blank, this will display the Fit Date.  If Fit Date is also blank, the date the history was entered will be displayed.
    3. If a Hearing Aid or Earmold, the Side will be shown here.
    4. The current Status of the device.
    5. The Manufacturer and Model name of the device.
    6. The Style of the device (if entered).
    7. The POS Invoice number linked to the device, if it has been auto-invoiced.

    Hearing Aid History Summary

      

    • Blue buttons indicate active options.
    • Orange buttons indicate that information has been entered and/or is outstanding (such as an Order).
    • Gray buttons indicate inactive/unavailable options.
    • Right hearing aids and earmolds are indicated with a pink header.

    • Left hearing aids and earmolds summaries are indicated with a blue header.

    • Accessories are indicated with a gray header.

    • Details for the hearing aid history, such as Battery Size, Serial Number, Purchase Date, etc will be displayed on the summary.  These details can be be changed only by clicking the Edit icon .  See below for a description of each field.
    • The Options field can be displayed or hidden by clicking the arrow. When options have been selected the word Options will turn orange and the options will be displayed. To add or edit options, click the Edit icon .

    • Notes can be displayed or hidden by clicking the arrow. When notes have been entered the word Notes will turn orange and the notes will be displayed.  To add or edit notes, click the Edit icon .

    • Repairs and L&D - can be displayed or hidden by clicking the arrow. If repairs have been entered, this label will turn orange.  

    If the repair is still outstanding it will display Outstanding Repairs and L&D.  

    See Adding a Repair for details.

    • Implant Components - can be displayed or hidden by clicking the arrow. If repairs have been entered, this label will turn orange.   See Implant Components for details.

    Hearing Aid History Details

    Click the Edit icon to view and edit details for the Hearing Aid History.

           


    Manufacturer

    The manufacturer of the device. Required.

    Model

    The device model. Models are filtered based on the selected Manufacturer. Once selected and saved, it can be changed in certain cases (not invoiced yet, status of On Order or Purchased Elsewhere) if this feature has been turned on for the practice.  Contact TIMS Support for details. Required.

    Third Party

    This box will be checked automatically if the HA History is a third party purchase.

    Invoice

    Check this box to create a POS invoice for this device.  The next time the POS module is run for this patient, you'll be asked if you want to include the hearing aid on the invoice.  Once a hearing aid history has been invoiced, it cannot be deleted.  (This will be automatically checked for new orders or stock items, but can be unchecked if needed).  See Automated Invoicing for more information.

    Credit

    Check this box to create a POS credit memo for this hearing aid.  (This will be automatically checked for returned aids that have been previously invoiced) See Automated Credit Memos for more information.

    Style

    Will be automatically copied from the model setup, but can be changed if needed.

    Battery Size

    The type of battery used in the selected device. Once a Manufacturer and Model are selected this field will automatically display the default Battery size for the model. You can change the size if you wish.

    Status

    The Status of the hearing aid history will allow you to track each stage of a sale and/or fitting.  Certain default statuses are used in TIMS, and cannot be deleted or changed. Required.

    Options

    The options associated with the hearing aid. Click on the Options button to choose the desired options for the aid.

    Serial Number

    The serial number of the device. If a stock item has been selected, this will be automatically filled in from the stock record.

    UDI Number

    The Unique Device  Identifier of the device. If a stock item has been selected, this will be automatically filled in from the stock record. This field will be visible only if turned on for the practice (contact Support for details). 

    Vendor Invoice Number

    If known, the number of the invoice received from the manufacturer.  If a stock item has been selected and an invoice number has already been entered when the item was received, this will be automatically filled in from the stock record.  If an invoice number was entered in the Order screen, this will automatically be filled in from there.

    Seen By

    The provider who sold/fit the device. The Seen By will default to the provider of the last appointment for the patient.  If there is no previous appointment,  it will default to the provider selected in the Patient  details.  If there is no default there, you can add one here by clicking the field.  Depending on the practice setup, the Seen By may not be editable after the record has been saved.

    Purchase Date

    The date the device was purchased by the patient. This is the date used in Opportunity Tracking.

    Current Warranty Date

    The date that the warranty on the selected device expires.  If a default warranty period has been entered for the selected Model, then this date will default to that many months after the entered Purchase Date. The date can be changed if needed.

    Warranty Type

    If a special warranty type applies to this device, it can be selected here. (default is Manufacturer)

    Original Warranty Date

    The Warranty Date of a hearing aid may sometimes need to be adjusted for various reasons, such as a repair, special warranty purchase, etc. If the Current Warranty date is changed, then the original date is preserved in this field, and cannot be edited.

    Fit Date

    The date the selected device was fit and received by the patient.

    Loss/Damage Date

    The date that the Loss and Damage Warranty expires. 

    Loss/Damage Type

    If a special Loss and Damage warranty type applies to this device, it can be selected here.

    Is CROS

    Check this box if the hearing aid is used to treat a single sided hearing loss. 

    Price

    The price that will be charged for the device.  If it was entered as a stock item, the price will default to the stock price.  If added as a new order, it will default to the price entered in the Model Setup.  In either case, the price can be edited. If the device is then auto-invoiced, this price will be used by default on the POS Invoice.

    Discount Type

    If discount will be given when the device is billed, you can select one here. Available discounts will be those that have already been entered in POS Setup.  If the discount is a percentage type, the amount will be calculated automatically and entered into the Discount field.  If a flat rate type is chosen, the amount entered in setup will be entered, but can be edited if needed.

    Discount

    If an existing percentage discount item is chosen in Discount Type, this will be calculated automatically and cannot be edited.  If a flat rate Discount was selected, this amount can be edited.

    Net

    The net price that will be charged for the device: Price - Discount

    Use Net on Invoice

    If checked, when the device is auto-invoiced the net price, including discount will be used as the item amount.  If unchecked, 2 line items will be used - one for the device, and one for the discount.

    Notes

    A free form field for miscellaneous notes about the device - up to 4000 characters can be entered.

    Tracking


       Invoice

    If the hearing aid has been invoiced in the POS module, the invoice number will be displayed here.

      Credit Memo

    If the hearing aid has been returned and a credit memo issued in the POS module, the credit memo number will be displayed here.

      Updated

    The date when the HA History information was last updated, along with the User and Site (location). This is automatically updated when the record has been changed and saved.

      Created

    The date when the HA History information was originally created, along with the User who created it.


    Delete a Hearing Aid History


    To delete a Hearing Aid History, click the Delete icon .  A confirmation message will popup before deleting.

     

    If you delete a Hearing Aid History that is for a Stock Item, you will prompted to either return the stock item for resale, or to inactivate it with reason entered.  You can not delete a history that has been invoiced, or has a custom order.


    Reports for Selected Hearing Aid History

    Click on the Reports icon to run patient and hearing aid history reports. User access to reports is assigned in user groups set up. 



    If the user has permission, the Hearing Aid Pricing Panel will show the hearing aids available for sale, the number currently in stock, and the default price.
