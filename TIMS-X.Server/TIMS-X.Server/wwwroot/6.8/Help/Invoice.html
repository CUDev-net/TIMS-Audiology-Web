  1. POS

    Invoice

    The Invoice screen contains the line items that were added to the invoice, along with Command buttons to add, edit, delete, print, and save.


    Select Source of Document

    Click New to create a new Invoice.  Click Open to select from a list of the patient's non-finalized invoices.  Click All to select from a list of all the patient's invoices (finalized or open).

    Patient

    The name and active address of the selected patient.

    Bill to Customer

    Will default to the patient’s name and active address in the Patient details. If

    an address is entered in Name & Address for Invoicing - this will be the default used.

    All fields can be edited as desired, and will be saved with the invoice only.

    Payment Type

    If a payment is to be entered onto the invoice, you can choose a type here, which will determine further processing after the invoice is finalized.  The default is Manual.

    Document Information


     Document Date

    Date of the invoice. Will default to the current date, but can be changed as needed.

     Site

    This will default to the site specified in the Patient details, but can be changed as needed.

     Provider

    Name of provider patient was seen by, who made this sale.  This is a required field. If you are invoicing a hearing aid from the HA History link, the Seen By will default to the Seen By in that link.  If you are not invoicing a hearing aid, it will default to the provider of the last appointment for the patient.  If there is no default, you can select the provider by clicking the field.  If there is a previous appointment and depending on the practice setup rules, the Provider cannot be changed. This information will be exported to the Sales Rep field in the QuickBooks® invoice. 

     Appointment

    The last appointment for the patient.  Used to determine the Marketing Source and for various outcome reporting. This will default to the patient's last appointment, but can be changed if needed.

     Marketing

    The Marketing Source will default to the source of the last appointment for the patient.  If there is no previous appointment, or the last appointment did not have a source - you can select one here by clicking the field.  Once the source is set and the invoice is saved and depending on the practice setup rules, it cannot be changed.  This information will be exported to the corresponding field in the QuickBooks invoice.

     Tax Rate

    Specifies tax group for the invoice. Will default to the tax group chosen as “Default” in Setup.

    Line Details


     Item

    The sale Item name - either a POS Item, HA Model, Battery Size, or CPT Code

     Description

    Description of the item. Will default to the description entered in setup. This field can be edited as needed.

     P.O. No.

    If the item is an auto-invoiced hearing aid, this will automatically be filled in

    with either the P.O. No. from the device order, or the P.O. No. from the stock item when it was received.  This field is not editable.

     Unit Price

    Price per unit. Will default to the price entered in the Setup. Can be edited as

    needed.

     Units

    Quantity or units of items, can be edited as needed.

     Amount

    Total amount for the given item; automatically calculated from the quantity and the unit price.

     Tax

    If the line item is taxable, this box will be checked. This will default to the taxable status entered in the Setup. Taxable status can be changed by highlighting the line and clicking the Edit button. (or double-clicking on the line) 

     Clm

    If the line item will also be billed as a claim (insurance transfer), this box will

    be checked. To edit, highlight the line and click Edit, or double-click on the line.

    Click the down arrow next to Expand to Enter Billing Codes for Insurance Transfer. This only be available if a previous Appointment has been selected.

    Click the Enter Billing Codes button . Choose one CPT code, at least one Diagnosis Code (maximum four), and an optional Modifier (maximum four) from the code lists. Click OK.  Enter an amount to be billed to insurance (default is the entire line item amount, but can be edited to a lesser amount). See Insurance Transfers for more information.

     3rd Party

    If the item is a Third Party auto-invoiced hearing aid, this box will be checked.

    This field is not editable.

    Document Totals

    The subtotal (without tax), tax total, total invoice amount, and remaining balance (including other applied payments) of the invoice.

    Office Notes

    Open field for internal notes only.  These will not be printed on the customer's receipt, or exported to QuickBooks®.

    Customer Message 

    Customer message that will print on the invoice. Messages can be added in Setup.

    Memo

    Open field for memo notes. These can be included on a printed document if desired, and will be exported to QuickBooks.

    Line Commands

    • Add - add a new line item to the invoice
    • Insert - insert a new line item  to the invoice above the highlighted line
    • Edit - view and/or edit details for the highlighted line item 
    • Remove - deleted the highlighted line item from the invoice.

    Document Commands

    • Final - lock the invoice so that it can no longer be edited.  Only finalized invoices will be exported to QuickBooks.
    • Print - click once to open a preview window.  Click the print icon on the preview to print to printer.   Note: Printing an open POS invoice will automatically finalize it.
    • Documents - this button will only be active if the invoice has been finalized.  Click here to run reports for the active patient or invoice.
    • Delete/Void - If the invoice is still Open, this button will display "Delete".  If finalized, it will display "Void". Deleting an invoice will remove it entirely from the database.  This is only allowed with an open invoice.  A finalized invoice can only be Voided.  An invoice that has been exported to QuickBooks cannot be deleted or voided.
    • Apply POS - not active for invoices
    • Close - close the active invoice, but remain in the POS Invoice module.  
    • Exit - close the POS module entirely
