    Open Transactions Mode

    In this mode, newly entered transactions that have not yet been generated into a claim are displayed on the patient tree.

    Patient Tree

    Expanded:

    Collapsed:

    If the tree is collapsed, expand by clicking on the arrow symbol next to the patient name.  

    1. The First Level of the tree will display the Patient Name.  
    2. The Second Level displays the Type of transaction, Appointment Bill To Provider, and Appointment Site.
      1. New - newly added transactions that have not been submitted to any payer yet
      2. Pri - transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the primary payer.
      3. Sec - transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the secondary payer.
      4. Ter - transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the tertiary payer.
      5. Quar - transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the quaternary payer.
      6. Quin - transactions that have been previously generated into a claim, sent to the clearinghouse, adjudicated and resent to Claims Processing link to resubmit to the quinary payer.
    3. The Third Level of the expanded tree shows the CPT code of the transaction, Date of Service, and Charges entered.  Hovering the mouse over the line will popup a tooltip displaying the CPT code description.

    Select one or more transaction lines to see the details in the Claim Information section to the right. Select by clicking in the checkbox to the left of the line.


    Only the transaction lines can be edited here - double-click on a line to open the Claim Transaction detail window.  Here you can add, edit or view transaction information.

    Action Buttons

    The following buttons appear on top of the tree when in Open Transactions mode:

    • Expand - expands all nodes on the tree, so that all levels are displayed for all patients.
    • Collapse - collapses the tree to a condensed state, so that only the top level (patient name) is displayed
    • Select All - places a check mark next to all transactions in the tree
    • UnSelect - removes the check mark from all transactions in the tree
    • Create All - generates all selected (checked) transactions into claims.  If no lines have been selected, no action will be performed.
