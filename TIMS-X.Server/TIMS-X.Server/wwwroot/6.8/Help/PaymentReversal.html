    Payment Reversal

    A Payment Reversal is a negative value payment used to either correct another payment entered in error, or to represent a refund to the patient.


    Create a new Payment Reversal using the same steps as for a regular Payment Receipt. Refer to the saving and printing information under the that section for more clarity. You cannot finalize a payment reversal with an amount of zero.


    Select Source of Document

    Click New to create a new Payment.  Click Open to select from a list of

    the patient's non-finalized payment receipts.  Click All to select from a list of

    all the patient's payment receipts (finalized or open, regular or reversals).

    Payment Receipt Type

    Click either the Payment or Refund Check box.  This will determine which options will be available. 

    A Payment Reversal (negative payment) can be entered to correct a payment entered in error (for a wrong patient for example).  This would generally be done when the prior payment has already been exported to QuickBooks® and cannot be edited, deleted or voided. A payment reversal cannot be applied to an invoice or an existing payment, however a credit memo or other payment can be applied to a reversal.  It will appear in the QuickBooks  Patient Register as type "General Journal" and as a negative amount in the deposit report, dated as the date of export from TIMS. (see QuickBooks instructions for applying the reversal in QuickBooks)


    A Refund Check would represent an actual refund issued to the patient.  It will appear in the QuickBooks  Patient Register as either type "General Journal" or "Check", depending on the account type chosen.

    Patient

    The name and active address of the selected patient.

    Payment from Name and Address

    Will default to the patient’s name and active address in the Patient details. If

    an address is entered in Name & Address for Invoicing - this will be the default used.

    All fields can be edited as desired, and will be saved with the invoice only.

    Payment Type

    Choose a type here, which will determine further processing after the payment receipt is finalized.  The default is Manual. Only available if type is Payment (not Refund Check).

    Document Information


     Payment Date

    Date of the payment reversal or refund check. Will default to the current date, but can be changed as needed.

     Site

    This will default to the site specified in the Patient details, but can be changed as needed.

    For Pmt Reversal


     Payment Method

    Choose from the payment methods previously entered in Setup. This is a required field.

     Payment Reference

    Check or Reference number.  This is required field.

     Payment Amount

    Total amount of the payment to be reversed.

     Rev

    Check this box to post this payment as a reversal.

     Applied Amount Remaining

    Not applicable for reversals.

    Payment applied to Claim?

    Not applicable for reversals.

    For Check Refund


     Refund Payable Accnt

    Choose from the Refund Check Accounts previously entered in Setup. This is a required field.

     Check No

    Check or Reference number.  This is required field. (but can be changed later if needed, even after finalization)

     Check Refund Amount

    Total amount to be refunded.

    Applied Amount Remaining

    Not applicable for refunds.

    Office Notes

    Open field for internal notes only.  These will not be printed on the customer's receipt, or exported to QuickBooks®.

    Customer Message 

    Customer message that will print on the invoice. Messages can be added in Setup.

    Memo

    Open field for memo notes. These can be included on a printed document if desired, and will be exported to QuickBooks.
