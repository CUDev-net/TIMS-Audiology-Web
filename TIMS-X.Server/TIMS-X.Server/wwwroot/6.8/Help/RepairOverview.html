    Repair Overview

    The Repair section contains the repair information for the selected hearing device. Repair procedures are the same for Hearing Aids, Earmolds, and Accessories.


    • Repair information can be hidden or displayed by clicking the arrow.

    • "Repairs and L&D" heading appears in black when a hearing aid history includes no repairs. 
    • "Repairs and L&D" heading appears in orange when a hearing aid history includes only closed repairs.
    • "Outstanding Repairs and L&D" heading appears in orange when a hearing aid history includes an open repair. 
    • Repair information displayed includes the date it was received from the patient, a checkbox to indicate if the repair will be invoiced, and the date the repair was returned to the patient.

    Repair Details

    View and edit repair information by double clicking on the repair date line.


    Repair Order Number

    A non-editable field that records a unique sequence number assigned to the repair.

    Payable

    This will be checked or unchecked by default depending on the Warranty Date. To change, check or uncheck the box.  The payable status is also seen in the repairs list, by displaying either Yes or No. If the repair is marked as payable, the next time the POS module is run for this patient, you'll be asked if you want to include the repair on the invoice.

    Repair Item

    Beginning in version 6.07, you can now select the Repair POS item that will auto-invoiced.  This field is only visible when the Payable box is checked.  See POS Item setup for details. 

    Create Loaner

    Check this box to automatically create a loaner hearing aid record when the repair is saved.

    Received from Patient

    The date the device is received for repair. The Received is automatically entered from the current date when a new repair is added.  It can be changed as necessary - click on calendar icon to choose the desired date. Required.

    Complaint

    The reason the repair is needed. Repair complaint reasons can be added or edited in Setup.

    Sent to MFG

    The date the device was sent to the manufacturer for repair.  

    Due Back

    The date the aid is scheduled to return from the repair facility. Required.

    Shipping Number

    A text field for entering the shipping number, if available.

    Back from Repair

    The date the aid was received back from being repaired. 

    MFG Invoice Number

    The number of any invoice received from the repair facility.

    Repair Action

    The action that was taken to repair the hearing aid - sending to vendor, repairing in-house, reprogramming, etc. This is a free form field, where up to 50 characters can be entered.

    Repair 

    Replacement Received

    L&D Replacement Received

    This box can only be changed once Back From Repair date has been entered.

    • Repair - checked by default, regular repair
    • Replacement Received - check if a replacement device has been received from the manufacturer
    • L&D Replacement Received - check if a replacement device has been received from the manufacturer for a Loss & Damage claim

    Replacement Serial Number

    Enter the serial number of the new hearing aid, if a replacement has been received.  This field will only be editable if the Replacement Received box has been checked.

    Replacement UDI Number

    Enter the UDI number of the new hearing aid (if used), if a replacement has been received.  This field will only be editable if the Replacement Received box has been checked.

    Repair Warranty Date

    The new warranty date for the repairs done on the device.  If this date is changed, you will be prompted to copy it to the Current Warranty date of the HA History if desired.

       

    Returned to Patient

    The date the repaired aid was returned to the patient.  Once this date is filled in and the repair has been saved, the Repair is no longer considered Outstanding.

    Repair Notes

    A free form field for miscellaneous notes about the repair.  Up to 1000 characters of notes can be entered to record any special circumstances.

    Tracking


      Invoice

    If the repair has been invoiced in the POS module that number will be displayed here.

      Updated - By User/By Site

    The date when the Repair information was last updated, along with the User and Site (location). This is automatically updated when the record has been changed and saved.

      Created  - By User

    The date when the Repair information was originally created, along with the User who created it.
