  1. QuickBooks®

    Setup and Maintenance

    QuickBooks


    Have QuickBooks® installed, updated and registered according to the instructions provided with the software. If you are in multi-user mode, switch to single-user mode and login with administrator rights.


    Note: If you currently work with an existing QuickBooks  database and/or are a Canadian, Australian, New Zealand, or UK user, please contact Support prior to any export.

    TIMS Software


    Maintenance


    1. Duplicate Patient Names - TIMS can be configured to either allow duplicate patient names (first, name, initial) or not.  The default is "no" (off).  This setting in TIMS determines how names will be exported to QuickBooks , as QB does not allow for duplicate names.
      1. Allow Duplicates is OFF  -  Duplicate patient names can not be entered in TIMS. Any existing duplicate Patient names must be changed or deleted.  If a duplicate name is found during the updates, the name will not be imported. The Duplicate Names report in the Reports-Patients menu can assist you in finding any duplicate names.
      2. Allow Duplicates is ON - Duplicate patient names can be entered in TIMS, but will be exported to QuickBooks with a Customer Name of Last, First Initial + ID#

    All patient names will be exported in this way if duplicates are allowed. Note: Only the Customer/Job name will have the ID number appended in QB.  The actual First, Last, and Initial will appear normally on invoices, statements, etc. 

    1. Any duplicate hearing aid model names, battery sizes, or CPT codes must also be changed or deleted prior to importing. The Duplicate Item Names report under Reports - Sales can be used as an aid.



    Note: For users importing into an existing QuickBooks database: If there are existing patients and item names in your QuickBooks  database, TIMS will attempt to match them up when syncing for the first time. The names must be exactly the same for a match to be made. For example Doe, John T in TIMS would be matched with Doe, John T in QuickBooks, but would not be matched with Doe, John or Doe, John S, etc.
