  1. What's New

    What's New in Version 6.08

    Patient Reports Designer

    • The Report Designer now uses advanced software tools to give you even more control over the design and layout of patient diagnostic reports.  Reports can be designed and merged/viewed in the same screen.
    • The Report Designer is now located under the Analytics/Reporting link.
    • Microsoft® Word is no longer required to design and run diagnostic reports and fax cover sheets.
    • Bookmarks have been categorized and reorganized so that they are easier view and add to reports.
    • Additional bookmarks have been added, including Acoustic Reflex data, Tympanogram data, audiometer setup information, and patient insurance.
    • Images such as audiograms, logos, and signatures can be viewed as their actual size in design mode.  There is no longer a need to merge to see how they will fit on the report.
    • Signatures can now be collected on diagnostic reports. New form fields, such as check boxes, drop-down selection fields, and date pickers can also now be added.   
    • Track Changes - If a report is saved as "Editable at launch", Track Changes can be turned on when the report is run. If the reports are run and saved as a draft, edits and comments can now be tracked by user.  Changes can be accepted or rejected and previous user comments can be viewed.  The final version of the report can then be saved to the patient record.
    • Diagnostic reports (both new and archived) can now be directly faxed and emailed to physicians or a one-time address/fax number.    

    Communications

    • Letters for both bulk communication and single patients also utilize advanced software tools making it easier to create custom letters. Microsoft® Word is no longer required. Letters can be designed and merged/viewed in the same screen.
    • Additional filters have been added for HA History, Orders, and Battery Club.
    • Additional bookmarks have been added for patient information.
    • Signatures can now be collected on letters. New form fields, such as check boxes, drop-down selection fields, and date pickers can also now be added on single patient letters.   
    • Communication letters can now be designed and used as Patient Intake Sheets.  They can either be run in bulk for a date range or singly per patient.  These customized intake sheets can be linked to the Intake Sheet button in Appointment Summary so that they can be printed with one click.
    • Letters (both new and archived) can now be directly faxed and emailed to physicians or a one-time address/fax number.

    Reports

    • Stock and Custom reports can now be set as "Favorites" from the TIMS Reports menu, with a simple Right Click - Add to Favorites.  The favorite reports can then be accessed from the user's Home link.  This is a per-user setting so that each user can have their own favorites and run them quickly from the Home screen.
    • New Reports
      • Appointment - Appointment Schedule Condensed
      • Claims - ERN Transactions
      • Data Extracts - HA History Export
      • Data Extracts - History Export
      • Management - Required Field Exceptions
      • Patient - Patients by Custom Field (formally Patients by Custom Text Fields)
      • Referral Source - Physician Search
      • Selected-Patient - Hearing Aid Quote (Patient) 

    Patients

    • All archived reports, letters, and documents (scanned or uploaded) can now be exported into a single .zip file where they all can be extracted to a folder.  This function is set by user permission and can also be tracked in the Audit Center.
    • Patient Activity Panel - several categories have been renamed for better clarity. Patient Reports is now named Archived Reports. Patient Documents is now named Imported Documents.
    • Updated CareCredit™ financing interface.  Apply for financing, make purchases and refunds, and view patient history directly from TIMS.

    Accounts Receivable

    • Patient Accounts Receivable data can now be generated in the TIMS reports screen, without having to go to a separate Accounts Receivable link.  The current A/R date can now be seen at the top of the reports menu.
    • The A/R generation can now be done in the background while you continue to work in TIMS in other areas.
    • A recurring task can now be setup to automatically generate A/R data on a set schedule.  Contact TIMS Support for details.
    • 2 additional GL accounts have been added to assign the default account for Earmolds and Accessories separately from Hearing Aids.
    • GL Export and Deposits have been moved to the Centers-Billing link.
    • The Deposits screen now includes a Date Range, to make loading payments faster.

    Appointment Notifications

    • Confirmations and Verifications can now be configured to be sent automatically on a pre-determined schedule. Contact TIMS Support for details and assistance in setting this up.
      • Select how many days in advance to send the notifications - either by Email, SMS, or Voice.  The task can also be filtered by Site, Provider, and Appointment Type.
    • Provider Notification Templates can now be copied from another provider, making setup easier if templates are similar. 

    Appointments

    • Online Patient Scheduling - practices that are replicated or cloud-based can now offer patients the ability to request an appointment online.  A unique and secure URL address will be generated that can be used to display a link on the practice website, or used in emails and texts.  Patients can request appointments on any device - phone, tablet, or computer. They will be able to select a location and specific (or any) provider, and availability will be shown in real time.
    • Appointments have been redesigned for increased performance and better speed.
    • Resources can now be turned off if the practice does not use them.
    • The Web Scheduler's technology platform has been upgraded and modernized.

    History

    • Scripted Notes - additional bookmarks have been added for Test Results, Previous History conditions, and patient insurance information.
    • Notes Designer - the editor and spell-checker for Chart Notes, Diagnosis, and Recommendation has been improved. It now includes even more word processing formatting and functions such as inserting images, charts, and text boxes.
    • Locked histories can now be made inactive if you do not wish them to be part of the patient's record. For example, if notes were made on the wrong patient and then locked they can not be deleted or edited.  With this new feature they can be made inactive so that they do not appear incorrectly on activity reports or chart notes.

    Hearing Aid History

    • Components can now be copied to the other side of a binaural hearing aid history.

    Claims

    • An easier way to submit corrected claims has been added.  You can now "copy" a claim directly in adjudication, make necessary changes, combine CPT codes if necessary, and add other required codes without having to switch screens.
    • In Adjudication, the upper and lower sections of the screen can now be collapsed - making it easier to see all of the claim transaction detail when there are many codes on the claim.
    • Two new modes have been added to Adjudication - Patient Responsibility and Negative Balance.
    • A new function has been added to generate a batch file that can be exported to a selected location on your computer. This can be used for situations where a batch of claims needs to be manually uploaded to a claims processing clearinghouse.  

    Setup

    • Stock Item Status - custom status can now be added for Stock on Hand devices
    • Gender - custom patient Genders can now be added, along with a choice of pronouns to be used in scripted notes.
    • Questionnaires - a duplicate of an existing "in use" questionnaire can now be made without inactivating the original. 
    • User Group - an easier interface has been added to add and remove permissions from a group.  All permissions can be added or removed with one button, or multiple can be selected at a time with Ctrl+Click.
    • Practice - Patient Type can now be set as a required patient field. A disclaimer field has been added that can be inserted at the beginning of all emails sent from the practice.

    Centers

    • Bounced Emails
      • The Bounced Emails Center is the place to view previously emailed documents and messages that have been returned because of an invalid email address.
    • Digital Intakes
      • This new center displays submitted digital intake forms that need to be reviewed.  Double-clicking on the patient name allows for reviewing and accepting or rejecting changes.
    • Auditing
      • User logins can now be tracked, viewed, exported and printed from the Audit Center. Sign-outs are tracked as whether they are regular, forced by inactivity, or forced by another user. Sign-ins are tracked as whether they are regular or MFA (Multi-Factor Authorization).
      • Claims and claim transactions edits are now tracked and included in auditing.
      • The new feature to export patient documents is included in auditing.

    Multi-Factor Authentication

    • For added system security, users can now be assigned to require Multi-Factor Authentication when logging in.  An MFA code can be sent to either a mobile phone number or email address. (fees applicable - contact TIMS Support for details)

    APIs – Application Programming Interfaces

    • Access a wealth of Audiology practice data for 3rd party marketing firms using standard REST/APIs. Select from a broad range of data categories - Practice, Location, Patient, Appointment, Hearing Aid sales and returns.
