    Transaction Lines and Details

    The Transaction Lines and Details section contains the Transaction lines that were added for the appointment, along with Command buttons to add, edit, delete, and save.




    After a Transaction Line is added, the applicable information can be edited by double-clicking on the line.


    From - To Date

    The From and To dates that the procedure was performed, services were rendered, or supplies were furnished for the transaction line. These will both default to the date of the appointment.  The To date cannot be before the From date. Claims cannot be submitted with a From date in the future.

    Place of Service

    The Place where the procedure was performed, services were rendered, or supplies were furnished for the transaction line.  The default is "11 - Office", but can be changed as needed.

    Rev

    Used for Institutional claims only.

    CPT Code and Description

    A code from the uniform classification system used to identify the procedure performed, service rendered, or supplies furnished.

    Modifiers

    A modifier code from the HCPCS uniform classification system used to identify special circumstances associated with the corresponding procedure code entered.  Additional modifiers can be displayed in the Additional Details screen.

    Diagnosis Codes

    A code used to identify that the provided service was treatment for one or more of the reported diagnosis codes.

    Note: A maximum of four Diagnosis Codes can be added per appointment.

    Unit Price

    The charges that apply to the CPT Code. The Unit Price will default to the price entered for the CPT Code in the Setup link.  

    Units

    The number of units associated with the CPT Code on the Transaction Line.

    Charges

    The total charge for this procedure, calculated by multiplying the Unit Price by the Units.  This field is not editable.

    Transaction Type

    If the transaction has just been created and not yet been submitted as a claim, this field will be "New". If it has already been submitted at least once and is being re-submitted to the payer it will display "Pri" or "Sec".  If the transaction has been created from a POS Insurance Transfer, it will display "New-pos".


    Command Buttons


    Line Commands

    • Group Add - click this button to add multiple transaction lines at once
    • Add - click this button to add a new transaction line
    • Edit - click this button to edit the selected transaction line
    • Combine - click this button to combine 2 insurance transfer lines into one line
    • Delete - click this button to delete the selected transaction line

    All Lines

    • Save - click this button to save your changes to all transaction lines associated with the appointment
    • Cancel - click this button to discard all changes made to the transaction lines, since the last save.
