    User Preferences

    Preferences can be set in two locations.  A user can access their own preferences from the Home link. Click the Set User Preferences button.  

    Or, if you are logged in as an Administrator, preferences can be accessed from the User Setup screen, by clicking the User Preferences button.


    Reset User Name

    If you want to change the User's name, type the new name here.

    Change Password

    Click this button to reset the User password.  The Change Password screen will appear.  The new password must be typed exactly the same twice in order to be saved.  Passwords must follow the rules setup in the Password Security section of the Practice Setup.

    Reset Window Layouts

    If you've pinned or docked the various windows in a way that becomes difficult to

    navigate, you can reset the panel and windows to the default settings.

    Associated Site

    If you select an associated Site, each time the user logs in, that Site will appear

    automatically in the drop down field. (it can be changed, if needed)  If a default

    Site is not chosen, the user will need to select one each time they log in.

    Default Start Link

    Select the link you wish for TIMS to go to on startup, or choose Last Visited to

    always start on the last link visited before exit.

    Default Font and Size

    Select the font and size to be used as a default in fields that allow for Rich Text

    formatting, currently History Link - Diagnosis, Recommendation, and Chart Notes. The default is Arial 9, if something different is not selected.

    Alert Snooze Duration (minutes)

    Set the number of minutes to re-display an alert after clicking the Snooze button.

    (default is .5 = 30 seconds)

    Alert Refresh Interval (seconds)

    Set then number of seconds to check for new alerts (default is 10 seconds)

    Communications Page Size

    If set to 0, no paging will be done on generated Communications Call and Letter lists. For larger databases that may have performance issues, set a number greater than zero to control the number of patients that will appear on each page. This will improve speed.

    Expand Task Tree by Default

    Check this box to keep the Task tree expanded all the time.

    Use Legacy Imaging

    If the user is having compatibility issues with their scanner and the imaging functionality, checking this box may help resolve them.

    Enable Spell Checker by Default

    Check this box to enable the built-in spell-checker for notes fields.

    Popup tasks on Login

    Check this box to have alerts popup for uncompleted tasks when the user logs in.

    Expand HA History Options and Notes

    Check this box to expand HA History notes and options by default.

    Show Completed Tasks by Default

    Check this box to show completed user tasks in the Home link by default.

    Clear Patient List when Existing Communications

    If the user is having issues with program slowness when generating large patient lists in Communications, checking this box will help resolve them. It is unchecked by default.

    Keep Imaging Open After Doc Save

    Check this box to have the imaging window stay open after saving a newly imported document.  This will allow you to print, fax, or email directly from the imaging window right away.


    Appointment Calendar Settings


    Calendar Time Interval

    Choose the increment for the calendar grid in the Appointment link.

    Calendar Colors From

    Choose the color-coding priority for the Appointment grid.  Appointments can be color-coded by Appointment Type, Site, or Resource.


    Copy History Forward Defaults

    Select the individual history/SLP fields that you wish to carry forward by default from previous to new history records.


    Expand Patient Activity Tree Defaults

    Select the individual Activities that you wish to see expanded by default on the Patient Activity Tree.


    Require Multi-Factor Authentication

    Check this box to require an MFA code when the user logs in. When the box is checked, additional fields will appear below.  You must enter either a Mobile Phone number or Email address where the user will receive the code. 

    Work Hours  (Select Site first)


    Office Hours

    Select the starting and ending hours that the user is available for appointments

    for each day of the week at each site.
